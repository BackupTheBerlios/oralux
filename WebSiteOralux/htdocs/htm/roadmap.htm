Roadmap

The following lines describe the main remaining points to reach Oralux 1.00. This release will offer a ready to use audio desktop for GNU/Linux users.

Oralux 2.00 is more futuristic, and will be an intuitive, ready to use audio desktop.

We wish to follow a humble and pragmatic approach. First with Oralux 1.00, targeting users who know GNU/Linux or who are able to learn it. Release 2.00 will more concern the persons who have no particular skill to use a computer, whereas the computer would be useful for them.

Only Oralux 1.00 is detailed here.

At the moment, the more urgent points are:

  • Oralux on HD. Two solutions will be offered: copying the compressed image on the hard disk, or a talking installation.
  • Graphical environment

Scenario

Oralux 0.6 alpha assumes that the user has a well detected sound-driver and is interested in a text-based environment. Other scenarios will be managed.

A consistent Text to Speech, Braille solution in a text-based environment.

  • Software Text to Speech (TTS) required, OSS or ALSA driver found.
  • Software TTS required, no OSS or ALSA driver available: PC speaker.
  • TTS and Braille required.
  • Braille required.
  • Hardware TTS with or without Braille.

A talking tty2, tty3,...

Useful for applications not compliant with Emacspeak or if tty1 is no more available (e.g. a software hangs).
